# ✅ Финальное решение для Referer: https://www.flip-trip.com/

## Из скриншота видно
**Referer:** `https://www.flip-trip.com/` (выделен фиолетовым)

## Что нужно добавить в Application Restrictions

### Вариант 1: С /* (рекомендуется)
```
www.flip-trip.com/*
```

### Вариант 2: Без /* (попробуйте, если первый не работает)
```
www.flip-trip.com
```

### Вариант 3: Оба варианта (для надежности)
Добавьте ОБА:
- `www.flip-trip.com/*`
- `www.flip-trip.com`

## Полный список доменов для добавления

Добавьте ВСЕ эти варианты:

1. `www.flip-trip.com/*` ← **это главное!**
2. `www.flip-trip.com` (без /* - попробуйте и этот)
3. `flip-trip.com/*` (без www)
4. `flip-trip.com` (без www и без /*)
5. `*.flip-trip.com/*` (wildcard)
6. `*.fliptripfrontend.vercel.app/*`
7. `fliptripfrontend.vercel.app/*`
8. `*.vercel.app/*`
9. `localhost:5173/*`

## Критически важно проверить

### 1. Places API (New) в API Restrictions

1. Google Cloud Console → **APIs & Services** → **Credentials**
2. Откройте ваш API ключ
3. В разделе **"API restrictions"** нажмите на выпадающий список
4. Убедитесь, что **"Places API (New)"** отмечен галочкой
5. Если его нет - найдите и добавьте его
6. Сохраните

### 2. Время применения изменений

После добавления доменов:
1. Нажмите **"SAVE"**
2. Подождите **минимум 5-10 минут** (иногда до 15 минут)
3. Сделайте **Hard Refresh** (Cmd+Shift+R или Ctrl+Shift+R)
4. Очистите кеш браузера (или используйте режим инкогнито)

### 3. Проверка после изменений

1. Откройте консоль браузера (F12) → **Network**
2. Найдите запрос к `/place/photo`
3. Проверьте **Status Code** - должен быть **200**, а не 403
4. Если все еще 403:
   - Проверьте, что домен добавлен правильно
   - Убедитесь, что Places API (New) включен
   - Подождите еще 5 минут

## Если ничего не помогает

### Альтернативное решение: Прокси на бэкенде

Если ограничения по доменам не работают, можно создать endpoint на бэкенде для проксирования фото:

1. Создать `/api/place-photo-proxy`
2. Фронтенд запрашивает фото через этот endpoint
3. Бэкенд использует серверный ключ (без ограничений по доменам)
4. Бэкенд возвращает фото фронтенду

Это обходит проблему с Application Restrictions, но требует дополнительной разработки.
